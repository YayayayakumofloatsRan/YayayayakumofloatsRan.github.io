\section{整体说明}
主页成品部署在 GitHub Pages，地址为 \href{https://yayayayakumofloatsran.github.io/}{https://yayayayakumofloatsran.github.io/}。 本项目基于笔者的工作项目与艺术偏好展开，以个人的天文观测经历作为主线组织了大量个性化素材进行视觉效果的展示。成品完成了个人介绍、数据可视化、彩蛋三个基本模块。介绍、项目说明等方面则从个人简历中改编而来；数据可视化部分则由笔者的绘画、摄影、电影欣赏等方面提供支撑；彩蛋方面，笔者鼓励读者去自行探索，包括初始界面的 3D 横向界面滑动设计、 DARK/LIGHT 的整体配色区分、 微型太阳系模型的相位/速度交互，同样在一个相对明显的为止有一个可交互的 apt deadlock meme， 读者可以自己去尝试。

笔者主要的 Coding 工具为 Windows 端 Codex gpt-5.4/5.5 xhigh， 其负责主要框架的编码与整体代码审查，笔者负责进行具体的翻页结构、交互设计、文字、素材内容的提供与修正要求。

\section{设计理念与视觉方案}

笔者的核心视觉追求是网页的表现力，避免使用纯色或过于繁琐的图案设计，Nemo Qi 为笔者英文名。本网页的视觉呈现是在迭代中不断产生的。用户头像被删除，月球图片中的地点信息经过公开化处理，画作中的个人签名也做了弱化。主页默认使用深色模式，指定深色以靛蓝和青色为主，金色只用于细线和重点边框。浅青色模式保留为可切换状态。

页面分为两段。第一段是横向 3D deck，包含 Intro，About，Projects 和 Astronomy 四张卡片。卡片使用显式边缘按钮翻页，鼠标滚轮留给纵向阅读。这个调整来自实际体验，滚轮翻页会破坏阅读节奏，也会让卡片难以停在屏幕中央。第二段从 Gallery 开始恢复传统纵向阅读，用于放置绘画素材，电影数据，Notes 和 Contact。

首屏使用月球图像作为视觉锚点，phase = 0.885 的月面照片天然地提供了一个良好的主页表现方案，蕴含着天文的明线主题。Astronomy 区展示月面，SKYWATCHER 200mm F5 牛顿反射镜，极轴镜视角和太阳系控件。Gallery 和 Movies 使用同一套图片卡片与 inspector 说明区，文字不直接压在图片上。这样能保证图片本身干净，访问者停留或点击时再获得解释，避免了每张图片的解释说明堆砌。

\textbf{踩坑复盘} 1.第一类问题是隐私问题，一旦给它提供了简历参考，个人主页容易被写成完整简历副本，这对于公共网站而已越界明显，如默认头像、地点和签名等（包括后来添加的摄影、绘画素材）都有必要及时剔除，在参考阶段就不应该向 codex 提供。笔者在多轮反馈中删除头像，只保留 Nemo Qi 这个公开称呼，并把月球图中的地点与画作中的签名做马赛克处理，最后部署前同样人工核验了信息发布的去隐私化。

2.早期迭代阶段，同样遇到主次关系分布把握不到位的问题，在起初要求尚不完整的情况下，codex 擅自加入了大量空洞的说明性文字，没有很好执行笔者对其“留空”的要求，同样，codex 添加的项目同样与笔者的真实重心不匹配，这属于笔者早期心理预期尚不明确时的疏忽，没有提供项目示例。后续调整为 Holo 主项目，ProjectH 只作为研究方向出现，公开页面不展开私人仓库细节。项目介绍也因此变短，保留链接和位置即可。

3.交互节奏。横向 3D 卡片的视觉效果较强，但滚轮触发会影响正常阅读。多次指出卡片无法停在屏幕中央、侧边和底部滚动条突兀、内部滚动条破坏观感。最终方案改为边缘按钮翻页，按钮平时近似隐藏，用户触碰或聚焦时才出现，保留横向切页的立体感，也让纵向阅读恢复稳定。早期的版本渲染优化并不好，控件徒有其表，却会频繁触发，造成翻页上的卡顿，此间进行了大量的控件设计调整，优化了事件流。

4.图片及其说明。最初的 Gallery、Movies 和 Astronomy 把注释压在图上，画面被文字遮挡。后续改为 inspector 说明区，鼠标停留或键盘聚焦时更新说明。这个调整让图片先作为视觉素材出现，解释文字再按需出现，视觉负担明显降低。迭代初期，笔者与 codex 进行了大量可行的排版尝试，由于 codex 并没有真正的审美能力，它对于网页空间的布局很多时候同人的直观审美感受是违背的，例如人会追求图片、文字的上下镶嵌的效果，并且可以直观看出二者分布比重的协调与否，但又没有方法直观想象出合适的排布，所以就需要大量的调整尝试，并非 codex 的执行不到位，往往是在 prompt 阶段尚且对成品没有定型的预期，所以对于每次的迭代结果并不满意。笔者因此与 codex 对 Gallery,Movies 这些图片展示区做了大量的细节调整提示。

5.素材准确性。Astronomy 区曾出现月球图片重复、设备名称标注错误（如极轴镜、牛反这些）、太阳系动画比例粗糙等问题。后续按照素材文件夹标注替换图片，把设备名修正为 SKYWATCHER 200mm F5，并用行星公转周期设置速度比例。图像区域也从松散排列改为 3 行 2 列，太阳系控件单独占位，避免被挤压。Movies 区同样有类似的问题，codex 的图像识别工具调用可以让它轻易识别出笔者提供的大部分电影素材，但仅让它自行处理效果不理想，需要标注以节省时间。

6.图片预览和电影截图。点击图片后打开大图的初版反应极慢，且只对少数图片生效。后续把预览绑定到整张卡片，优先打开遮罩，再加载高清图。Movies 区的截图也从裁切显示改为居中保留构图，Contact 这类宽银幕画面不再偏到一侧。

7.部署。GitHub Pages 初次访问时图片加载失败，原因是发布根目录和资源目录不一致。后续加入根目录跳转、Pages 工作流和 .nojekyll，资源路径才稳定。公网部署还要检查发布入口和资源相对路径，具体的部署工作可以交给模型代工，但是人必须要起到充分的监管作用，并且提供人视角下的反馈，否则codex随时可能曲解用户意图。
