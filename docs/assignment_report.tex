\section{整体说明}
主页成品部署在 GitHub Pages，地址为 \href{https://yayayayakumofloatsran.github.io/}{https://yayayayakumofloatsran.github.io/}。本项目基于笔者的工作项目与艺术偏好展开，以天文观测经历作为视觉主线，组织公开项目、绘画素材、摄影素材和电影偏好。页面完成了个人介绍、项目入口、电影偏好统计和交互彩蛋等基本模块。彩蛋包括首屏 3D 横向切页、DARK/LIGHT 主题切换、微型太阳系相位与速度控制，以及一个可触发的 apt deadlock meme。

笔者主要使用 Windows 端 Codex gpt-5.4/5.5 xhigh 完成 Coding。模型负责框架搭建、代码实现、资源接入和部署修复；笔者负责确定公开边界、筛选素材、修正文案、评价视觉效果，并通过多轮反馈收敛页面结构。

\section{设计理念与视觉方案}
笔者的核心视觉追求是让主页有足够强的表现力，同时避免纯色平铺或繁琐花纹。Nemo Qi 为笔者英文名。页面默认使用深色模式，深色以靛蓝和青色为主，金色用于细线、按钮边界和重点标识；浅青色模式保留为可切换状态。用户头像被删除，月球图片中的地点信息经过公开化处理，画作中的个人签名也做了弱化。

页面分为两段。第一段是横向 3D deck，包含 Intro、About、Projects 和 Astronomy 四张卡片；第二段回到纵向阅读，放置 Gallery、Movies、Notes 和 Contact。横向卡片只通过边缘按钮切换，鼠标滚轮保留给纵向阅读。这个调整来自实际使用：滚轮翻页会破坏阅读节奏，也会让卡片难以停在屏幕中央。

首屏使用 phase = 0.885 的月面图像作为视觉锚点。Astronomy 区展示月面、SKYWATCHER 200mm F5、极轴镜视角和太阳系控件。Gallery 与 Movies 使用同一套图片卡片和 inspector 说明区，说明文字不直接覆盖在图片上，访问者停留或点击时再获得解释。

\subsection*{踩坑复盘}
第一，隐私边界需要前置。个人主页很容易被写成完整简历副本，头像、地点、签名和本地路径都会提高公开风险。最终页面只保留 Nemo Qi 这个公开称呼，并对月球地点和画作签名做公开化处理。这个问题提醒笔者，素材进入公网前要检查图像本身是否带出额外信息。

第二，项目主次需要反复校正。早期页面对次要项目着墨过多，与笔者真实重心不匹配。后续调整为 Holo 主项目，ProjectH 只作为研究方向出现，公开页面不展开私人仓库细节。报告中也只保留 Holo 链接，不展开技术细节。

第三，交互节奏要服从阅读。横向 3D 卡片视觉效果较强，但滚轮触发会影响纵向浏览。最终方案改为边缘按钮翻页，按钮在未触碰时近似隐藏，用户聚焦或触碰时出现。这个处理保留了横向切页的立体感，也让正文阅读恢复稳定。

第四，图像说明要与图像分离。初版 Gallery、Movies 和 Astronomy 把注释压在图上，画面被文字遮挡。后续改为 inspector 说明区，图片先作为视觉素材出现，解释文字再按需出现，视觉负担明显降低。

第五，素材准确性必须人工核验。Astronomy 区曾出现月球图片重复、设备名称标注不准、太阳系速度比例粗糙等问题。后续按照素材文件夹标注替换图片，把设备名修正为 SKYWATCHER 200mm F5，并按行星公转周期设置速度比例。Movies 区也需要人工标注，以节省模型识别素材的时间。

第六，部署路径不能只用本地打开来判断。GitHub Pages 初次访问时图片加载失败，原因是发布根目录和资源相对路径不一致。后续加入根目录跳转、Pages 工作流和 .nojekyll 后，资源路径才稳定。公网发布需要同时检查页面入口、资源路径和 Actions 状态。

\section{总结}
本次网页制作从个人主页框架开始，逐步收敛为以天文视觉、公开项目、绘画素材和电影偏好组成的个人 blog。笔者的主要工作是持续判断真实偏好、公开边界和视觉问题；模型的主要作用是快速实现页面结构、交互逻辑、素材接入和部署修复。最终页面可以公网访问，报告也保留了足够清晰的截图证据和调试记录，后续仍可继续扩展 Notes 与公开文章。
