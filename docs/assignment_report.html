一、成品截图

截图为本地最新版本首页首屏。页面已部署到 GitHub Pages，根路径会直接打开成品站点。

二、设计说明

这份主页想呈现一个公开的个人工作台。Holo 是主轴，天文观测提供视觉气质，电影和画作留作兴趣记录。默认主题采用浅青色；深色主题使用靛蓝与青色，金线只作点缀，让页面有夜空感。

首屏使用横向 3D deck。四张卡片从 Intro 推到 Astronomy。继续向下滚动时，Gallery 到 Contact 回到普通阅读流。这个结构让交互集中在首屏附近，后面的内容保持安静。

三、作业要求对照

要求实现方式
自我介绍区About 卡片写明公开身份，补充学校与专业方向，并给出邮箱。月球图像承担视觉主线。
真实小数据可视化Movies 区把 8 部 favourite movies 按主题分组并计数，形成小型条形图；下方卡片墙展示对应电影。
交互彩蛋页面可以切换明暗主题；apt 按钮显示终端彩蛋；太阳系控件负责天文交互。图片预览负责素材查看，电影筛选和画廊筛选提供入口。
公网部署已推送到 GitHub Pages 仓库，并通过 GitHub Actions 发布 site/ 目录。

四、关键 Prompt 记录与修改思路

Prompt 1：“做个人主页，深色系或靛蓝+金线；不放头像，只写 Nemo Qi。”
作用：先确定隐私边界和视觉方向。后续改为默认浅色，深色主题保留夜空感。
Prompt 2：“把 Holo 放到主项目位置，补充天文经历，写明数理兴趣，并放入 ProjectH。”
作用：明确项目主线。Projects 页围绕 Holo 展开，其他项目作为补充。
Prompt 3：“主页横向展开，像 Apple 切换屏幕；部分内容保留上下滚动。”
作用：形成现在的横向 3D deck 和纵向阅读流。后续取消滚轮翻页，只保留边缘按钮，浏览更稳定。
Prompt 4：“素材文件夹里有 astronomy/gallery/movies；图片点击要大图预览；orbit 做得更真实。”
作用：导入真实素材，统一 Gallery 和 Movies 的图片比例，并给太阳系控件加入周期比例。
Prompt 5：“对照作业三要求，文案微调，报告收工。”
作用：最后补齐作业对照，压缩 Holo 文案，并把 Movies 明确为小数据展示。

五、踩坑复盘

1. 一开始内容太像技术说明书

AI 容易把个人主页写成工程文档，句子长且机械。后续多次删减文字，只保留能体现个人兴趣和项目主线的内容。

2. 图片注释曾经遮挡图片

最初三个图片区把文字压在图上，观感差。后来改用 inspector 区域展示说明，图片表面保持干净。

3. GitHub Pages 图片一度加载失败

当时 GitHub Pages 仍从仓库根目录渲染 README，/assets 路径因此 404。后来改用 GitHub Actions 发布 site/，并保留 root index 和 .nojekyll 兜底。

4. 隐私边界需要反复检查

素材来自本地文件，页面很容易泄露本机路径和拍摄地点。最后用脚本扫描敏感字段，并对月球图和画作签名做了公开化处理。

六、如果重做会怎么改 Prompt

如果重做，我会在第一轮 Prompt 里写清作业 rubrics：自我介绍要可见，小数据要有图，交互要能操作，报告和部署要一起收尾。这样能更早约束页面结构，减少后期返工。
