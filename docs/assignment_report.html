作业三：Vibe Coding 个人主页报告

作品：Nemo Qi Personal Blog / 数据小馆

线上链接：https://yayayayakumofloatsran.github.io/

一、成品截图

截图为本地最新版本首页首屏。页面已部署到 GitHub Pages，根路径会直接打开成品站点。

二、设计说明

我想呈现的是一个偏“个人实验室”的主页，而不是模板化简历页。页面围绕 Holo、天文观测、量化研究、绘画素材和电影偏好展开，用浅青色默认主题配合深色天文主题，强调“系统、观测、小数据”的个人特征。

首页采用横向 3D 卡片展示 Intro、About、Projects、Astronomy，后续 Gallery、Movies、Blog、Contact 保持传统纵向浏览。这样能保留一点交互感，同时不影响普通阅读。

三、作业要求对照

要求实现方式
自我介绍区About 卡片写明 Nemo Qi、学校、专业、方向、公开邮箱，并用月球图像和天文主题代表个人兴趣。
真实小数据可视化Movies 区使用我给出的 8 部 favourite movies，按 Space / Music / Mind / Myth / Velocity 分类计数，做成轻量条形分布图和卡片墙。
交互彩蛋包括明暗主题切换、apt 终端彩蛋、太阳系 orbit 控件、图片大图预览、电影和画廊筛选。
公网部署已推送到 GitHub Pages 仓库，并通过 GitHub Actions 发布 site/ 目录。

四、关键 Prompt 记录与修改思路

Prompt 1：“做一个个人主页，深色系或靛蓝+金线，不要头像，不要过于 private，写 Nemo Qi。”
作用：确定隐私边界和最初视觉方向。后续我要求默认浅色，深色保留靛蓝、青和金线。
Prompt 2：“主要项目是 Holo，不是 laser_extraction；加入天文、Fermi/Feynman、数理兴趣、ProjectH。”
作用：把项目主次改正确，避免主页像普通项目列表。Holo 被设置成 Primary project。
Prompt 3：“主页横向展开、像 Apple 切换屏幕；一部分横向，一部分传统上下滑动。”
作用：形成当前的横向 3D deck + 纵向内容流结构。后续我又要求取消滚轮翻页，只保留边缘按钮，解决浏览体验问题。
Prompt 4：“素材文件夹里有 astronomy、gallery、movies；图片点击要大图预览；orbit 做得更真实。”
作用：把真实素材导入 site/assets，统一 Gallery 和 Movies 的图片比例，加入太阳系周期比例和 lightbox。
Prompt 5：“对照作业三要求，文案微调，报告收工。”
作用：最后把 Movies 明确改成真实小数据可视化，简化 Holo 文案，补齐报告结构。

五、踩坑复盘

1. 一开始内容太像技术说明书

AI 容易把个人主页写成工程文档，句子长且机械。我通过不断要求“简短、个人特色、宁缺毋滥”把 About、Projects、Blog 改成更像个人主页的表达。

2. 图片注释曾经遮挡图片

最初 Gallery、Movies、Astronomy 把文字压在图上，观感差。后来改成 inspector 区域，图片保持干净，悬停/聚焦时更新文字说明。

3. GitHub Pages 图片一度加载失败

根因不是图片缺失，而是 Pages 还在用仓库根目录渲染 README，导致 /assets 路径 404。解决方法是 GitHub Actions 发布 site/，并加 root index 和 .nojekyll 作为兜底。

4. 隐私边界需要反复检查

素材来自本地路径，容易把来源路径、地点或过于私人的信息带进页面。最后用脚本扫描敏感路径和字段，并对月球图、画作信息做了公开化处理。

六、如果重做会怎么改 Prompt

我会在第一轮就把作业 rubrics 写进 Prompt：必须有自我介绍、真实个人小数据、交互彩蛋、报告结构和部署方式。这样能更早约束页面结构，减少后期返工。
